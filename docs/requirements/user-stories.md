## 1. 用户故事概述

### 1.1 故事分类
本文档包含以下类型的用户故事：
- **身份管理故事**：用户身份创建、切换、认证相关
- **首页功能故事**：个性化欢迎、学习路径选择、推荐内容
- **大师功能故事**：大师搜索、详情查看、预约服务
- **课程功能故事**：课程浏览、学习、评价
- **社群功能故事**：动态分享、评论互动、圈子管理
- **个人中心故事**：个人资料、学习管理、收入统计

### 1.2 故事优先级
- **P0**：核心功能，MVP必须包含
- **P1**：重要功能，影响用户体验
- **P2**：增强功能，提升体验
- **P3**：优化功能，锦上添花

## 2. 身份管理用户故事

### 2.1 用户注册与身份创建

#### US-001: 用户注册
**优先级**: P0  
**用户类型**: 新用户  
**故事描述**: 作为新用户，我希望能够注册账号，以便使用Master Guide平台的服务。

**验收标准**:
- 用户可以通过邮箱和密码注册
- 注册时需要选择主要身份类型（大师/学徒）
- 注册成功后自动登录并跳转到首页
- 注册失败时显示具体错误信息

**技术要点**:
- 前端：Vue 3 + Element Plus表单验证
- 后端：Go + Gin处理注册逻辑
- 数据库：PostgreSQL存储用户基础信息

#### US-002: 创建大师身份
**优先级**: P0  
**用户类型**: 已注册用户  
**故事描述**: 作为用户，我希望能够创建大师身份，以便在特定领域提供指导服务。

**验收标准**:
- 用户可以选择专业领域
- 需要填写个人介绍和技能标签
- 需要上传资质证明文件
- 提交后进入审核状态
- 审核通过后身份生效

**技术要点**:
- 前端：文件上传组件、表单验证
- 后端：文件存储到OSS、审核流程
- 数据库：user_identities表存储身份信息

#### US-003: 创建学徒身份
**优先级**: P0  
**用户类型**: 已注册用户

### 2.2 身份切换与管理

**优先级**: P0

**故事描述**: 作为拥有多个身份的用户，我希望能够快速切换身份，以便在不同角色间无缝切换。
